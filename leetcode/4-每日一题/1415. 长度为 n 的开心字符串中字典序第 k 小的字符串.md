# 1415. 长度为 n 的开心字符串中字典序第 k 小的字符串

> **日期**：2026-03-14  
> **所用时间**：21min  
> **知识点**：DFS、回溯、字典序

## 1. 回溯

用 DFS/回溯按字典序生成所有长度为 n 的开心字符串：从空串开始，每一位在 `'a','b','c'` 中选一个，且必须与当前已构造串的最后一个字符不同。按 `'a' → 'b' → 'c'` 的顺序尝试，自然得到字典序递增的序列。收集到第 k 个时即可返回（或先全部收集再排序取第 k 个，代码中为后者）。

复杂度分析：

- **时间复杂度**：$O(3 \times 2^{n-1})$，即所有开心字符串的数量（首位 3 种，其余每位 2 种）。
- **空间复杂度**：$O(n)$，递归栈深度与构造的字符串长度。

**Python3**

```python
class Solution:
    def getHappyString(self, n: int, k: int) -> str:
        ans = []
        @cache
        def dfs(i, s):
            if i < 0:
                nonlocal ans
                ans.append(s)
                return
            for c in ['a', 'b', 'c']:
                if s == "" or c != s[0]:
                    dfs(i - 1, c + s)
        
        dfs(n - 1, "")
        if len(ans) < k:
            return ""
        return sorted(ans)[k - 1]
```
